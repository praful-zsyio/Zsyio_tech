1. Authentication
-------------------------------------------------------
POST /api/login/
Description: Login - Obtain Access and Refresh JWT tokens.
Allowed Emails: praful@zsyio.com, abhishek@zsyio.com, ashutosh@zsyio.com, 
                ayan@zsyio.com, arpit@zsyio.com

Payload Option 1 (using login_id):
{
  "login_id": "praful@zsyio.com",
  "password": "your_password"
}

Payload Option 2 (using username):
{
  "username": "praful@zsyio.com",
  "password": "your_password"
}

Response (Success):
{
  "refresh": "eyJ0eXAiOiJKV1QiLCJhbGc...",
  "access": "eyJ0eXAiOiJKV1QiLCJhbGc...",
  "user": {
    "id": 1,
    "username": "praful@zsyio.com",
    "email": "praful@zsyio.com",
    "first_name": "Praful",
    "last_name": "Sonwane",
    "full_name": "Praful Sonwane"
  }
}

Note: If first_name or last_name are not set in the database, they will be 
      empty strings. The full_name will be constructed from first_name and 
      last_name, or fallback to username if both are empty.

Response (Error - Invalid Credentials):
{
  "detail": "Invalid credentials. Please check your email and password."
}

Response (Error - Unauthorized Email):
{
  "detail": "This email is not authorized to log in."
}

POST /api/login/refresh/
Description: Refresh Access token using Refresh token.
IMPORTANT: This endpoint does NOT require an Authorization header.
           Only send the refresh token in the request body.

Headers:
Content-Type: application/json

Payload (JSON):
{
  "refresh": "YOUR_REFRESH_TOKEN"
}

Response (Success):
{
  "access": "eyJ0eXAiOiJKV1QiLCJhbGc..."
}

Response (Error - Missing Token):
{
  "detail": "Refresh token is required.",
  "error": "Please provide a valid refresh token in the request body.",
  "example": {
    "refresh": "your_refresh_token_here"
  }
}

Response (Error - Invalid/Expired Token):
{
  "detail": "Invalid or expired refresh token.",
  "error": "The refresh token is not valid or has expired. Please login again."
}

Troubleshooting:
- If you get "Invalid character in header" error in Postman:
  1. Go to the "Authorization" tab and select "No Auth"
  2. Remove any Authorization header from the "Headers" tab
  3. Only send the refresh token in the Body (raw JSON)
- The refresh token should be the exact string from the login response
- Do not add "Bearer " prefix to the refresh token

Note: Old endpoints /api/token/ and /api/token/refresh/ are still available 
      for backward compatibility.


2. Projects
-------------------------------------------------------
GET /api/projects/
Description: List all projects.

POST /api/projects/
Description: Create a new project (Requires Auth/Admin).
Payload (JSON):
{
  "title": "E-Commerce Platform",
  "category": "Web Development",
  "summary": "A full-stack e-commerce solution.",
  "description": "Built using React and Django, features cart, payments, etc.",
  "image": "https://example.com/image.jpg",
  "tech_stack": ["React", "Django", "PostgreSQL"],
  "tags": ["Featured", "New"],
  "live_url": "https://myshop.com"
}

GET /api/projects/:id/
Description: Get project details. Example: /api/projects/1/

DELETE /api/projects/delete/<project_id>/
Description: Delete a project (Requires Auth/Admin).
Method: DELETE (Do NOT use GET)
Example: /api/projects/delete/1/


3. Services
-------------------------------------------------------
GET /api/services/services/
Description: List all services.

POST /api/services/services/
Description: Create a service (Admin).

GET /api/services/technologies/
Description: List all technologies.

POST /api/services/technologies/
Description: Add a technology (Admin).
Payload (JSON):
{
  "name": "React",
  "category": "Frontend",
  "icon": "react",
  "color": "#61DAFB"
}

DELETE /api/services/delete/<id>/
Description: Delete a service by ID (Admin).
Example 1: /api/services/delete/1/
Example 2: /api/services/delete/2/

DELETE /api/services/technologies/delete/<id>/
Description: Delete a technology by ID (Admin).
Example: /api/services/technologies/delete/1/
